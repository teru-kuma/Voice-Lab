VOICE LAB · 取扱説明書

はじめに

VOICE LAB は録音と文字起こしを同時に行うフィールドレコーダーです。録音データと文字起こしはすべてこの端末内に保存され、外部には送信されません(文字起こしと AI 機能の利用時のみ、それぞれの音声認識・LLM サーバーと通信します)。

基本の使い方

  • 画面下の大きな ● ボタンを押すと録音開始。もう一度押すと停止して保存されます。
  • 録音中は経過時間と波形が表示され、文字起こしがリアルタイムで流れます。
  • 保存されたメモは下の MEMOS 一覧に新しい順で並びます。

録音モード

録音を始める前に、中央の 3 つのチップで動作を選べます。

  • 録音+文字起こし — 音声とテキストの両方を記録(標準)。
  • 録音のみ — 音声だけ。文字起こしはしません。
  • 文字起こしのみ — テキストだけ。音声ファイルは残りません。

録音中はモードを変更できません。停止してから切り替えてください。

各メモの操作

  • ▶ — 音声を再生 / 停止。
  • COPY — 文字起こしをコピー。
  • EDIT — 文字起こしを手直しして保存。
  • AI ▾ — 要約 / 箇条書き / TODO抽出(要 AI 設定)。
  • SHARE — 端末の共有シートで他アプリへ送信。
  • SAVE .WEBM — 音声ファイルをダウンロード。
  • SAVE SET — 音声+文字起こしを zip でまとめて保存。
  • EXPORT ENGRAM — Engram 形式の Markdown で書き出し。
  • DELETE — メモを削除(確認あり)。

AI で処理した結果はカード内にアンバー色の枠で表示され、タップでコピーできます。

AI 機能の設定

ヘッダーの ⚙ から LLM を接続します。

  • PROVIDER — Anthropic(Claude)または OpenAI 互換(GLM 等)。
  • ENDPOINT — OpenAI 互換のときだけ入力(例: https://open.bigmodel.cn/api/paas/v4/chat/completions)。
  • MODEL — 使うモデル名。
  • API KEY — 各プロバイダの API キー。
  • ENGRAM PROJECT — エクスポート時の project 値(未設定なら inbox)。
API キーはこの端末のブラウザ内(localStorage)にのみ保存され、外部に送信されません。
OpenAI 互換エンドポイントはブラウザからの直接アクセス(CORS)を許可していない場合があります。ブロックされるときは Anthropic を使ってください。

即録音ショートカット

URL の末尾に &autorec を付けると、起動と同時に録音が始まります。

例: #m=both&autorec を開いた状態でホーム画面に追加すると、タップした瞬間に録音開始。事前にマイクを「常に許可」にしておいてください。

データの保存場所

  • メモは端末内のブラウザデータベース(IndexedDB)に音声ごと保存されます。ファイラーには現れません。
  • ファイルとして取り出すには SAVE .WEBM / SAVE SET を使うと Download フォルダに保存されます。
Chrome の「閲覧データを削除」でサイトデータを消すと、メモも消えます。残したい録音は事前にファイルとして書き出しておいてください。

同じアプリでも、開いた URL(localhost と GitHub Pages など)が違うとデータは別々に保存されます。使う場所は一つに固定するのがおすすめです。

Engram 連携

EXPORT ENGRAM は、AI 要約(あれば)を本文、生の文字起こしを Transcript として、frontmatter 付きの Markdown で書き出します。外部記憶 Engram の MEMORY 層に取り込む用途を想定しています。

うまく動かないとき

  • マイクが使えない — HTTPS または localhost で開いているか確認。マイク許可も必要です。
  • 文字起こしが出ない — 音声認識はネット接続が必要です。オフライン時は録音のみ動作します。
  • AI が失敗する — ⚙ の API キー / モデル名を確認。CORS ブロックの可能性もあります。
  • 録音が途中で切れる — 画面ロックや省電力設定の影響が出ることがあります。録音中はスリープを抑止していますが、端末側の設定が優先される場合があります。
